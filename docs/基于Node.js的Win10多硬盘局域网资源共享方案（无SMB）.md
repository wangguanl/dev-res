# 基于Node.js的Win10多硬盘局域网资源共享方案（无SMB）

## 一、核心方案定位

**核心目标**：Win10作为资源端（3块硬盘存储视频/音频/PDF），通过Node.js搭建Web服务，让局域网内的手机/PC通过IP访问所有硬盘资源，具备精美UI和资源分类能力。

**核心优势**：无需SMB协议，Node.js原生`fs`模块直接读写本地硬盘，配置简单、性能更高、无跨协议开销。

## 二、整体技术架构（分层设计）

|层级|核心技术/职责|
|---|---|
|资源层|Win10本地3块硬盘（D/E/F盘），存储视频、音频、PDF等资源|
|服务层（后端）|Node.js + Express：<br>1. 原生`fs`模块读写多硬盘文件；<br>2. 提供API接口（硬盘列表、资源分类、文件预览/下载）；<br>3. 跨域配置，支持局域网访问|
|展示层（前端）|Vue3 + Element Plus：<br>1. 响应式UI（适配手机/PC）；<br>2. 硬盘切换、资源分类标签；<br>3. 资源卡片展示、在线预览/下载交互|
|访问层|局域网内的手机/PC浏览器，通过Win10的局域网IP访问前端页面|
## 三、核心实现流程（分阶段落地）

### 阶段1：基础环境与核心配置（准备阶段）

1. **环境搭建**：

    - Win10安装Node.js（v16+），确保npm可用；

    - 搭建前端项目（Vite+Vue3）、后端项目（Express），安装核心依赖（cors/axios/Element Plus）。

2. **硬盘配置**：

    - 确认3块硬盘的Win10盘符（如D:/视频、E:/音频、F:/PDF）；

    - 后端配置硬盘白名单（指定需要共享的硬盘路径/文件夹，排除系统敏感目录）；

    - 放行Win10防火墙（允许后端端口3000、前端端口5173的入站请求）。

### 阶段2：后端核心功能开发（数据层）

1. **硬盘管理**：

    - 提供「硬盘列表接口」：返回配置好的3块硬盘名称+路径，供前端选择；

    - 权限校验：仅允许访问配置内的硬盘路径，防止非法访问。

2. **资源遍历与分类**：

    - 递归遍历选中硬盘的指定文件夹，获取所有文件的名称/路径/大小/修改时间；

    - 按文件后缀自动分类（视频：mp4/avi/mkv；音频：mp3/wav/flac；PDF：pdf；其他文件）；

    - 提供「分类资源接口」：接收前端传入的硬盘路径，返回该硬盘的分类资源数据。

3. **文件预览/下载**：

    - 流式传输文件：避免大文件加载到内存，支持浏览器直接预览（视频/音频/PDF在线播放）；

    - 断点续传（可选）：针对大视频，支持Range请求，提升播放体验。

### 阶段3：前端交互与UI开发（展示层）

1. **核心交互组件**：

    - 统一异常处理：文件不存在/权限不足时给出友好提示。

## 四、关键注意事项（避坑点）

1. **权限问题**：访问C盘/系统目录需以管理员身份启动Node.js，建议资源放在非系统盘（D/E/F）；

2. **路径编码**：文件/文件夹名称含中文时，前端传参用`encodeURIComponent`处理，避免乱码；

3. **性能优化**：多硬盘遍历耗时较长，添加缓存机制；大文件必须用流式传输，禁止一次性读取；

4. **外接硬盘适配**：若包含外接硬盘/U盘，拔出后程序仅提示「文件夹不存在」，重新插入即可恢复；

5. **跨域配置**：后端必须开启CORS，允许前端域名/IP访问，否则浏览器会拦截请求。

## 五、核心优势与落地优先级

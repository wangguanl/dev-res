# Node.js + Win10 多硬盘局域网资源共享方案（无 SMB）

## 1. 目标概述

**目标**：在 Win10 机器上，通过 Node.js 提供一个可在局域网内访问的资源共享服务，支持多硬盘（如 D/E/F）存储的视频/音频/PDF/文档等内容，并提供美观响应式 UI 供 PC/手机浏览器访问。

**核心特点**：
- 不依赖 Windows SMB/共享协议，使用 Node.js 原生文件读写能力
- 提供**硬盘白名单**与**路径权限控制**，避免越权访问系统敏感目录
- 支持**在线预览与下载**（视频/音频/PDF 预览，支持 Range 请求优化）
- 响应式前端（Vue3 + Element Plus / Vite）支持手机、平板、PC 访问

---

## 2. 适用场景与受益

### 2.1 适用场景
- 家庭/小型办公环境，需在局域网内共享本地多硬盘资源（影音、文档）
- 需要跨平台（iOS/Android/Windows/Mac）统一访问体验
- 希望绕过 Windows 网络共享配置复杂、权限问题以及性能瓶颈

### 2.2 受益点
- ✅ 简化配置：无需 Windows 共享、无需账户密码管理
- ✅ 访问速度更稳定：本地磁盘直接读取，无跨协议开销
- ✅ 安全可控：仅暴露白名单硬盘与目录，默认禁止访问系统目录
- ✅ UI 友好：分类、搜索、在线预览一站式体验

---

## 3. 架构概览

### 3.1 系统分层

| 层级 | 组件 | 角色/职责 |
|------|------|-----------|
| 资源层 | Win10 本地硬盘（D/E/F） | 存储影音/文档资源 |
| 服务层 | Node.js + Express | 提供 API、文件流、权限控制、跨域支持 |
| 展示层 | Vue3 + Element Plus（或其他轻 UI 框架）| 资源展示、交互、请求管理 |
| 访问层 | 局域网浏览器 | 端用户通过 HTTP 访问 |

### 3.2 网络架构

- 后端服务运行在 Win10 本地，监听**单个端口（建议 3000）**
- 前端可通过**静态资源托管**在后端也可单独运行（开发时 5173）
- 手机/PC 访问：`http://<Win10_IP>:<端口>`
- 后端启用 CORS：允许局域网内前端域名/IP 访问

## 4. 关键模块设计

### 4.1 配置与白名单

后端配置示例（`config.json` / 环境变量）：

- `PORT`: 服务端口
- `DISK_LIST`: 白名单硬盘及名称（示例：`[{"name":"视频","path":"D:/视频"},{"name":"音频","path":"E:/音频"}]`）
- `MAX_SCAN_DEPTH`: 目录遍历深度上限（避免扫描整个硬盘）
- `ALLOWED_EXTENSIONS`: 支持预览/展示的后缀列表

> ⚠️ 仅允许访问在白名单内的路径；所有输入路径必须进行标准化/校验，禁止使用 `..` 和任意路径跳转。

### 4.2 后端 API 设计

#### 4.2.1 硬盘列表
- `GET /api/disks`
- 返回：硬盘名称 + 根路径（按白名单）

#### 4.2.2 分类资源列表
- `GET /api/disks/:diskId/resources?category=video&folder=/xxx`
- 参数：
  - `diskId`：硬盘标识
  - `category`：分类标签（video/audio/pdf/all）
  - `folder`：可选，指定根目录下的子文件夹路径（相对路径）
- 返回：文件列表（name、relativePath、size、mtime、type、thumbnailUrl）

#### 4.2.3 预览 / 下载
- `GET /api/files/stream?diskId=&path=`
- 支持 `Range` 请求；流式传输避免一次性读全大文件

#### 4.2.4 搜索（扩展）
- `GET /api/search?q=&diskId=&category=`
- 通过名称模糊匹配查询资源

### 4.3 资源扫描与分类

- 读取白名单硬盘指定目录（如根目录或配置的子目录）
- 递归遍历但限制深度，避免耗时过长
- 使用扩展名映射为类别：
  - 视频：`mp4`, `mkv`, `avi`, ...
  - 音频：`mp3`, `flac`, `wav`, ...
  - 文档/PDF：`pdf` 等
  - 其他：其它格式
- 支持缓存：扫描结果可保存在进程内缓存或磁盘缓存，定期刷新

### 4.4 文件权限与安全

- 绝不允许任意路径访问；仅允许在白名单硬盘内访问
- 所有文件路径统一通过 `path.resolve()`、`path.normalize()` 进行标准化，并验证前缀
- 对于外接硬盘/挂载盘，异常（目录不存在）需返回明确错误，避免程序 crash

## 5. 前端设计（Vue3 + Element Plus 参考）

### 5.1 核心页面结构

- 顶部：硬盘选择下拉 + 分类 tabs
- 左侧（可选）：目录树视图（可折叠）
- 右侧：资源卡片列表（支持分页/加载更多）

### 5.2 主要交互

- 切换硬盘：请求 `/api/disks`，展示不同硬盘目录
- 切换分类：请求 `/api/disks/:diskId/resources?category=`
- 点击资源：
  - 支持在线预览（视频/音频/PDF 嵌入播放器/iframe）
  - 提供下载按钮（直接调用 `/api/files/stream`）

### 5.3 响应式布局

- PC：4 列卡片布局
- 平板：3 列
- 手机：1-2 列

### 5.4 错误与提示处理

- 目录读取错误（权限/不存在）：提示用户并建议刷新或检查硬盘插入
- 文件加载失败：展示友好提示并提供“重试”按钮

## 6. 部署与运维

### 6.1 本地测试

- 启动后端：`node server.js` 或 `npm run dev`
- 启动前端（开发模式）：`npm run dev`（默认 5173）
- 访问：`http://localhost:5173`

### 6.2 局域网访问

- 获取 Win10 IP：`ipconfig` → IPv4 地址
- 手机/他机访问：`http://<IP>:<端口>`

### 6.3 生产部署建议

- 使用 `pm2` / `systemd` 守护后端进程
- 将前端构建输出（`dist`）直接托管在后端，实现单端口访问
- 设置防火墙规则：仅允许局域网 IP 段访问端口

## 8. 风险与注意事项

- **管理员权限要求**：访问系统盘（C:）或受限目录时需以管理员权限运行 Node.js
- **性能瓶颈**：大目录遍历可能耗时，建议只扫描指定目录并使用缓存
- **权限验证**：必须严防路径穿越攻击（`..`、URI 编码绕过）
- **硬盘离线**：外接盘拔出或挂载变化应优雅处理，并提示用户
